# حاسبة مساحات الأراضي الزراعية — نسخة احترافية (Supabase + Netlify + GitHub)

## 1) ما الذي تغيّر عن النسخة الأصلية؟

| العنصر | قبل | بعد |
|---|---|---|
| البيانات | ملف CSV عام على GitHub (يقرأه أي شخص) | جدول `land_plots` في قاعدة بيانات Supabase، يُقرأ فقط بعد تسجيل الدخول |
| الدخول | لا يوجد | صفحة دخول احترافية (الاسم الكامل + الجوال/واتساب + البريد)، دخول لاحق بالبريد فقط عبر رمز تحقق (بدون كلمة سر) |
| التجاوب | جيد جزئياً | مُحسَّن بالكامل لسطح المكتب / التابلت / الجوال (جداول قابلة للتمرير أفقياً، خطوط متجاوبة) |
| الحفظ | لا يوجد | إمكانية حفظ كل حساب باسم مخصص واسترجاعه لاحقاً (مرتبط بحساب المستخدم) |
| النشر | غير منشور | جاهز للنشر على Netlify مرتبطاً بـ GitHub (نشر تلقائي عند كل تعديل) |

## 2) كيف يعمل التطبيق

- **`index.html`** — صفحة الدخول. بها تبويبان:
  - **حساب جديد**: الاسم الكامل، رقم الجوال (واتساب)، البريد الإلكتروني → يرسل النظام رمز تحقق من 6 أرقام إلى البريد.
  - **تسجيل الدخول**: البريد الإلكتروني فقط (لمن لديه حساب بالفعل) → نفس آلية رمز التحقق.
  - بعد إدخال الرمز الصحيح يتم تحويل المستخدم تلقائياً إلى `app.html`.
- **`app.html`** — التطبيق الفعلي (محمي، لا يفتح إلا بعد تسجيل الدخول):
  - يجلب بيانات الأحواض والمستأجرين من جدول `land_plots` في Supabase.
  - البحث بالاسم + فلترة بالناحية ورقم الحوض، مع اقتراحات فورية.
  - حساب المساحات بنظام فدان / قيراط / سهم (1 فدان = 24 قيراط = 576 سهم) وتحويلها لمتر مربع وسعر إجمالي حسب سعر الفدان المُدخل.
  - جدول "قطع تُطرح" لحساب الصافي بعد الخصم، مع تنبيه إن تجاوز المطروح الأصل.
  - حفظ أي حساب باسمه واسترجاعه أو حذفه لاحقاً (جدول `saved_calculations` — كل مستخدم يرى حساباته فقط).
  - طباعة/تصدير PDF عبر طباعة المتصفح.
- **`assets/config.js`** — مكان وضع مفاتيح الاتصال بمشروع Supabase الخاص بك.
- **`supabase_schema.sql`** — كل أوامر SQL اللازمة لإنشاء الجداول وصلاحيات الأمان (RLS).
- **`land_plots.csv`** — نفس بيانات `Zira.csv` الأصلية، مُجهّزة لاستيرادها مباشرة داخل Supabase.

## 3) خطوات الإعداد (بالترتيب)

### أ) إنشاء مشروع Supabase
1. أنشئ حساباً مجانياً على supabase.com وأنشئ مشروعاً جديداً.
2. من القائمة الجانبية: **SQL Editor → New query**، الصق محتوى ملف `supabase_schema.sql` بالكامل ثم اضغط **Run**.
3. من **Table Editor → land_plots → Insert → Import data from CSV**، ارفع ملف `land_plots.csv` المرفق (الأعمدة متطابقة تلقائياً).
4. من **Authentication → Providers**، تأكد أن **Email** مُفعّل، وفعّل خيار **Email OTP** (إرسال رمز تحقق رقمي) إن لم يكن مفعّلاً افتراضياً — موجود ضمن إعدادات Email في نفس الصفحة.
5. من **Authentication → URL Configuration**، أضف رابط موقعك على Netlify (بعد نشره في الخطوة التالية) ضمن **Site URL** و **Redirect URLs**.
6. من **Project Settings → API**، انسخ القيمتين: **Project URL** و **anon public key**.

### ب) تجهيز المشروع محلياً
افتح ملف `assets/config.js` وضع القيمتين اللتين نسختهما:
```js
const SUPABASE_URL = "https://xxxxxxxx.supabase.co";
const SUPABASE_ANON_KEY = "ey...";
```

### ج) رفع المشروع على GitHub
1. أنشئ مستودعاً (Repository) جديداً على GitHub، مثلاً باسم `land-calculator`.
2. ارفع كل ملفات هذا المجلد إليه (عبر GitHub Desktop أو الأوامر التالية):
```bash
git init
git add .
git commit -m "نسخة احترافية مع تسجيل الدخول وSupabase"
git branch -M main
git remote add origin https://github.com/USERNAME/land-calculator.git
git push -u origin main
```

### د) النشر على Netlify مرتبطاً بـ GitHub
1. سجّل دخولك على netlify.com باستخدام حساب GitHub نفسه.
2. **Add new site → Import an existing project → Deploy with GitHub**، واختر المستودع الذي رفعته.
3. اترك إعدادات البناء كما هي (لا يحتاج المشروع أمر بناء — Publish directory = `.`)، ثم اضغط **Deploy**.
4. بعد انتهاء النشر، انسخ رابط الموقع (مثل `https://land-calculator.netlify.app`) وأضفه في Supabase كما في الخطوة (أ-5).
5. من الآن، أي تعديل تدفعه (`git push`) إلى GitHub سيُنشر تلقائياً على Netlify.

### هـ) (اختياري) ربط دومين خاص
من لوحة Netlify: **Domain settings → Add a domain**، واتبع التعليمات لتوجيه الدومين الخاص بك.

## 4) ملاحظات أمان مهمة
- مفتاح `anon public key` آمن للاستخدام في الواجهة الأمامية، لكن كل الحماية الفعلية تتم عبر سياسات **RLS** الموجودة في `supabase_schema.sql` (كل مستخدم يرى بياناته فقط، وقراءة الأراضي للمسجّلين فقط).
- لا تستخدم أبداً مفتاح `service_role` داخل أي ملف يُنشر على المتصفح.
- الدخول بدون كلمة سر (عبر رمز تحقق بالبريد) يقلل مخاطر تسريب كلمات المرور، وهو ما اخترناه هنا كأفضل ممارسة لتطبيق بهذا الحجم.

## 5) تشغيل تجريبي على جهازك قبل النشر
يكفي فتح `index.html` مباشرة من أي خادم محلي بسيط (المتصفحات تمنع `fetch` من ملف محلي مباشرة أحياناً):
```bash
python3 -m http.server 8080
```
ثم افتح `http://localhost:8080` في المتصفح.
